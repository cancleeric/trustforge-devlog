參考資料：TrustForge 的方法與產業根基

TrustForge 的「多源信任提煉」並非空談——底下是把我們的演算法接上學術方法、以及實際打的真實來源與基礎設施。連結皆為公開可驗證資源。

一、學術方法 / 論文（信任發現 · 校準 · 反操縱）

  • Dawid–Skene (1979) — 最大似然估計觀察者錯誤率，眾源信譽推斷的奠基方法。我們以 EM 離線兜底實作動態信譽收斂。
    doi:10.1093/biomet/63.1.83 · Google Scholar
  • Truth Discovery / CATD (Li et al.) — 異質資料衝突和諧化的真理發現框架（Conflicts to Harmony）。
    Google Scholar
  • CRH (Yin et al., 2008) — 多衝突資訊源的真相發現（Copy-Resolved Truth Discovery 一脈）。
    Google Scholar
  • LTM — Latent Truth Model (Wang et al., KDD 2016) — 以潛變數建模來源可靠度。
    Google Scholar
  • Invest (Dong et al., PVLDB 2009) — 貝氏途徑從衝突來源發現真相，真理發現經典論文。
    Google Scholar
  • Confidence Calibration — Guo et al., ICML 2017 — 現代神經網路校准（On Calibration of Modern Neural Networks），我們的「資訊完整度」分層對齊此精神。
    arXiv:1706.04599
  • Manipulation / Coordination Detection — 協同文案與操縱風險識別的文獻脈絡。
    Google Scholar

二、產業大廠 / 真實資料源（我們實際接入或對齊的）

  • HOYA BIT（颯倍斯） — 黑客松命題方；官方 OHLCV 為我們的價格真值基準。
    hoyabit.com.tw
  • CoinGecko — 價格 / 情緒 / 市場數據 API（我們的多源行情與冷卻協調來源）。
    coingecko.com/en/api
  • SEC EDGAR — 美國證管會全文檢索（監管維度的真實申報來源，如 Bitcoin ETF / 上市公司持幣揭露）。
    sec.gov/edgar/search
  • Blockchain.com — 鏈上即時指標（手續費 / 難度 / 鏈上統計）。
    blockchain.com/explorer
  • Cointelegraph — 加密新聞來源（多源交叉比對之一）。
    cointelegraph.com
  • Reddit — 社群維度（經查證已終止 self-service API，TrustForge 明確排除，不卡流程）。
    reddit.com/dev/api

三、基礎設施 / 模型

  • AWS Bedrock — Claude Sonnet 4.6 — 語意矛盾判斷（僅當特徵，最終信任分由自家確定性公式產生，守反作弊）。
    aws.amazon.com/bedrock
  • GitHub Pages — 本開發記錄的靜態託管（entries.json 為客戶端可查詢資料集）。
    pages.github.com
  • SQLite — 開發記錄的來源真相資料庫（build_db.py 由 DB 生成靜態站）。
    sqlite.org

四、選定技術 / 技術堆疊（TrustForge 實際採用）

  • Python 3（後端，以標準庫為主） — 信任評分、多源攝取、Hermes 自主改進框架的核心。
    python.org
  • TypeScript + React（前端） — 專業金融情報儀表板（信任雷達 / 星系總覽 / 多幣總覽）。
    typescriptlang.org · react.dev
  • Vite + Vitest — 前端建構與單元測試。
    vitejs.dev · vitest.dev
  • pytest — 後端回歸 / 反作弊 / 成本鎖測試，並設覆蓋率下限。
    docs.pytest.org
  • AWS App Runner — 後端服務部署（apprunner.yaml）。
    aws.amazon.com/apprunner
  • AWS Systems Manager (SSM) + EventBridge — 部署防護閘與 Hermes cycle 定時排程。
    aws.amazon.com/systems-manager · aws.amazon.com/eventbridge
  • SQLite — 快取、永久成本帳本與租約、開發記錄來源真相。
    sqlite.org
  • nginx — 前端靜態服務與零停機切流（host-wide flock 防競爭）。
    nginx.org
  • Docker — 服務容器化（Dockerfile）。
    docker.com
  • GitHub Actions — CI（部署健康監控、release smoke、覆蓋率門）。
    github.com/features/actions

方法論上，我們不盲目套用單一論文：同源只算一票、資料不足就棄權不硬給數、動態信譽用先驗保留避免少數源被放大——這些都是把學術真理發現「誠實地」落到產品上的工程選擇。
