參考資料：TrustForge 的方法與產業根基

TrustForge 的「多源信任提煉」並非空談——底下是把我們的演算法接上學術方法、以及實際打的真實來源與基礎設施。連結皆為公開可驗證資源。

一、學術方法 / 論文（信任提煉的許多面向）

TrustForge 的「多源信任提煉」橫跨多個研究領域。以下按面向整理我們所參考／對齊的方法論。連結均為公開可驗證資源（arXiv / DOI / Google Scholar），未直接實作的方法也列出其脈絡，方便讀者追溯我們的設計依據。

🧠 核心（Core）才是我們的護城河：直接構成信任評分演算法的論文與真值來源——動態信譽收斂（Dawid–Skene）、多源交叉驗證、Bedrock 語意矛盾特徵，以及 HOYA BIT / CoinGecko / SEC / Blockchain / Cointelegraph 等真實資料本體。這一塊必須持續研究、越做越強，否則外框再漂亮也沒有競爭力。

🪛 外框（Outer）：運作、部署、UI、工具鏈（Python / React / SQLite / nginx / Docker / App Runner / CI）。它們讓核心能被穩定交付，但本身不含信任演算法。

📚 參考：方法論地圖、已排除項、待接入待辦——靈感與脈絡，非直接依賴。

圖例：✅ 已實作/上線 🔬 研究中(已寫未上線) 📚 方法論/概念參考 ⛔ 已排除

必要性（核心 vs 外框）：🧠 核心 直接構成信任評分演算法的論文/真值來源（我們必須持續研究變強的部分） ｜ 🪛 外框 運作/部署/UI/工具鏈，不含信任演算法本身 ｜ 📚 參考 方法論/已排除/待辦（非直接依賴）

八、待辦：接台灣監管源

對一個 HOYA BIT（台灣交易所）命題的產品，監管維度只接 SEC EDGAR 是缺口。下列為接入台灣對等來源的具體待辦（產品側執行，主 repo 處理；此處留作追蹤）：

  • ☐ 調查 MOPS 公開資訊觀測站 是否有可用數據介面（爬蟲/開放資料），對齊 SEC EDGAR 全文檢索做法
  • ☐ 盤點 金管會 FSC 虛擬通貨 VASP 名單與警示清單，作為「合規/風險」維度來源
  • ☐ 評估 TWSE / TPEx 公開揭露（上市櫃加密相關業務揭露）能否自動擷取
  • ☐ 接入 BlockTempo 動區 等台灣在地媒體 RSS，補新聞維度的在地來源
  • ☐ 建立「台灣監管」維度與既有多維信任雷達的對接（誠實標註資料不足時留白）
  • ☐ 驗證台灣源 live 模式確實打真實介面（沿用「證明 live 走真實源」的對稱降級日誌與 fail-closed 開關）
  • ☐ 更新本參考頁狀態：MOPS/FSC/TWSE 由 📚 建議接入 → ✅ 已接

方法論上，我們不盲目套用單一論文：同源只算一票、資料不足就棄權不硬給數、動態信譽用先驗保留避免少數源被放大——這些都是把學術真理發現「誠實地」落到產品上的工程選擇。
