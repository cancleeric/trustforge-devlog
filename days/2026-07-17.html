Day 18 · Isabella 接手開發日誌 + SkillHub-only 工作流 + 權限整理

2026-07-17

分類：基礎建設 ｜ 標籤：#isabella #devlog #skillhub #gitea #github #workflow

今天的主題不是新增一個炫目的功能，而是把 TrustForge 的「開發記錄工作流」正式接到可持續運作的軌道上：讓 Isabella 具備寫日誌、讀 repo、查 SkillHub、使用 Gitea / GitHub 線索、並誠實標註資料來源邊界的能力。這是把開發歷程從「人手回憶」推向「可追溯、可驗證、可交接」的一天。

1. 開發日誌 repo 接手

  • 確認公開日誌站來源：GitHub 上找到並 clone cancleeric/trustforge-devlog，確認這是 TrustForge 每日開發記錄的 GitHub Pages repo。
  • 確認目前最新公開日誌：站內最新一篇為 2026-07-16（Day 17），主題是 Hermes 持續智慧、升級治理控制平面與五年歷史回填。
  • 確認資料結構：日誌站採 entries.json 作為靜態資料索引，days/*.html 作為單篇頁面，feed.xml 與 sitemap.xml 由腳本重建。
  • 確認新增流程：使用 add_day.py 新增每日 HTML 頁、更新首頁索引、重建 Atom feed 與 sitemap。

2. Isabella 的開發工具與權限整理

  • Gitea / Wiki 存取修復：容器內的 Gitea / Wiki 教學文件確認位於 /opt/data/GITEA-WIKI-ACCESS.md；修正 HOME=/opt/data/home 導致 git / tea 讀不到原本認證檔的問題。
  • Gitea 驗收通過：tea whoami 回到 isabella，git ls-remote 可讀 hurricanesoft/wiki-docs，Gitea API 版本檢查成功。
  • sudo 權限確認：容器內 hermes 使用者已有免密碼 sudo，可用 apt-get 安裝缺少的開發工具，也可用 sudo 寫入 /usr/local/bin。
  • GitHub 現況誠實標註：公開的 trustforge-devlog 可讀；主專案 cancleeric/TrustForge 目前仍需要 GitHub 認證才能讀取，因此本篇不宣稱讀過私有主 repo 的新 commit。

3. SkillHub-only 工作流落地

  • 停用本機 skills：Eric 明確授權後，Isabella 將本機 skills 移出 active 目錄，避免再由本機 skill_view / skills_list 當主要技能來源。
  • 阻止重啟自動 re-seed：Hermes 重啟後曾自動補回 bundled skills；補上 /opt/data/.no-bundled-skills marker 後，確認 sync_skills 進入 opt-out。
  • 驗證 prompt 乾淨：重測 prompt builder，skills_prompt_len = 0、contains_available_skills = False、contains_skill_view = False。
  • SkillHub 成為單一技能來源：TrustForge 任務透過 mcp_skillhub_search_skills 搜尋，並用 mcp_skillhub_load_skill 載入 trustforge-continuous-analysis。

4. 對 TrustForge 的實際意義

  • 開發歷程不再只靠記憶：日誌 repo、entries 索引、feed、sitemap、SkillHub 技能與持久記憶被串成一條可重複的工作線。
  • 資料來源邊界更清楚：能讀到的公開 devlog 與 Gitea 線索會寫入；暫時讀不到的 GitHub 私有主 repo 不會被想像成已驗證事實。
  • 後續可半自動化：一旦 GitHub 主 repo 認證完成，就可用 draft_from_git.py 依每日 commit 自動整理草稿，再由 Isabella 潤飾成公開、可追溯的開發日誌。
