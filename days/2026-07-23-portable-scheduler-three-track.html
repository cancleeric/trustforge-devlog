工程意義

這輪 main 的價值在於把「TrustForge 在誰的機器上跑」從隱含假設改成可安裝、可渲染、可 rollback、可拒絕不受管理 collision 的明確營運契約。排程不再依賴 Eric 個人機器上的固定路徑，對未來在不同使用者、不同路徑、Linux user service 或 macOS launchd 間遷移都比較安全。

三軌學習文件則把 continuous analysis 的治理邊界補得更硬：歷史 RAG 回答只能是非證據型記憶，OHLCV 只能做 market baseline／outcome，不可冒充當時分析樣本；wrapper 候選只能進入 proposal、sandbox、review 與 human activation，不得自行修改 Trust Kernel、Evidence binding 或 production 狀態。

develop 的後續 78 個 commit 顯示產品線正在同時補資料契約、前端資料可讀性、指標 connector 與安全 scanner；但這些目前仍以 origin/develop 觀測為主，本篇不把它們宣稱為 GitHub main 已正式釋出。
