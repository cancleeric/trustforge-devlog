Day 19 · EC2 HTTPS CA runtime hotfix + v0.16.1 發版收斂

TrustForge 主線更新至 v0.16.1，針對 EC2 update-in-place 與 first-boot 部署路徑補齊 certifi，讓 fetch-scheduler／外部 HTTPS connector 在可攜式 CA trust store 下維持憑證與 hostname 驗證；本次驗證完成部署腳本字串斷言與 bash 語法檢查，但 pytest CLI 未安裝，完整 pytest 未能執行。

  • 分析時間：2026-07-18 03:01:26 UTC
  • Devlog repo：https://github.com/cancleeric/trustforge-devlog.git

主要變更

  • deploy/deploy_ec2.sh 的 EC2 update-in-place SSM 部署命令改為安裝 boto3 certifi，避免只在開發／打包環境有 CA bundle、但 EC2 執行環境缺少可攜式 trust store。
  • pyproject.toml 版本提升為 0.16.1，並將 certifi>=2024.2.2 納入正式 runtime dependency。
  • CHANGELOG.md 新增 v0.16.1，記錄 EC2 update-in-place 與 first-boot 路徑都需要補齊 certifi，以支援 fetch-scheduler probe 與 DynamoDB cache cost-ledger 驗證前的外部 HTTPS 連線。
  • tests/test_deploy_ec2_unit_no_credential_layer.py 新增迴歸檢查：部署腳本必須包含 pip3 install boto3 certifi，且不得退回舊的 pip3 install boto3 寫法。

工程意義

這次不是功能擴張，而是部署可靠度修補。TrustForge 近期已把外部 HTTPS connector 的憑證／hostname 檢查收緊；若 EC2 runtime 沒有明確安裝 certifi，安全策略會在部署後因缺少 CA trust store 而變成可用性風險。v0.16.1 把相依套件從隱含環境假設改為部署腳本與 package metadata 的明確契約，讓安全檢查與 EC2 啟動路徑一致。
