Day 2 · 第二階段 — 真資料上線：真連接器 + 快取 + 商業儀表板

2026-07-01

第二階段做一件最根本的事：讓 TrustForge 跑真實資料。信任提煉的價值在於「對多源資訊做可信度評分」——如果那些「多源」是假的示範資料，整個差異化就沒有意義。這一階段把產品從「離線示範」推進到「真連接器 → 真信任分析」，而且維持零額外成本。

1. 從示範樣本到真實多源

  • 產品現在直接接真實市場來源：加密新聞（RSS）、鏈上指標（如交易所餘額、恐懼貪婪指數）、社群情緒（論壇）、監管公告（SEC），加上命題官方提供的 HOYA BIT 五年價格資料。
  • 每一筆證據都是真的、可回溯的——來源名稱、時間戳、原始連結，而不是預先寫好的示範文字。
  • 信任層（來源信譽 × 交叉佐證 × 時效 − 操縱懲罰）現在作用在真實資料上：真實市場常有的「新聞看多、鏈上看空」這類跨源分歧，產品會如實呈現，而不是被抹平。

2. 又快又不燒錢：定時抓取 + 快取

  • 免費公開 API 不是無限量——狂打會被限流甚至封鎖。所以產品不在每次查詢時即時去打來源。
  • 改成背景定時抓取（各來源依其速率上限排程）→ 存進雲端快取（含自動過期）→ 使用者查詢時從快取讀取。既快、又不會把來源打爆。
  • 信任評分是產品自家的確定性演算法、不需大型語言模型；因此「真連接器 → 真資料 → 信任加權結論」這條路成本為零（語言模型只用於選用的文字潤飾）。

3. 商業等級的操作介面

  • 介面升級為專業金融情報儀表板（深色終端風）：左側查詢控制台、右側結果儀表。
  • 信任分數儀表 + 四分項拆解（信譽 / 佐證 / 時效 / 操縱），每一項旁邊都附一句話「為什麼」（像營養標示）——例如「命中 N 個獨立來源佐證」「48 小時內」；操縱項以扣分方式呈現，忠實對應演算法。
  • 跨源分歧卡：當獨立來源對同一議題立場相反時，並列對照、一眼看懂。
  • 資料透明：介面清楚標示本次分析用的是哪一種資料模式，讓每個結論都可溯源。

4. 交付與狀態

  • 版本 v0.4.0 已上線於固定公開網址，跑在真實資料上、對使用者零成本。
  • 官方要求的交付件（分析報告、可回溯證據清單、執行紀錄）皆由真實資料產出。

到這一步，TrustForge 不再是「示範」，而是一個能對真實加密市場的多源資訊做信任提煉的產品：真來源、真評分、可回溯、看得出分歧——這就是我們想交付、也想拿去用的東西。
