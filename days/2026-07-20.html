主要變更

  • HOYA BIT 與排程器 fail-closed 收斂：CHANGELOG.md 記錄 v0.16.15，HOYA BIT 即時真值來源在 endpoint 未設定、不安全或被停用時不產生佔位證據；排程器也不再把停用來源記成成功刷新，避免 cache/archive 被假成功污染。
  • 名人／鯨魚交易信號接入：新增 src/trustforge/ingestion/whale_trades.py、demo/sample_data/whale_trades.json 與 .kiro/specs/celebrity-whale-trades.md。Whale Alert／Arkham 來源採固定白名單 URL、5 秒 timeout、512 KB 上限，API key 只讀環境變數且不寫入 URL/meta/log。
  • 信任評分新增市場參考層：scoring.py 新增 whale_onchain=0.88 與 celebrity_trade=0.50，並給鯨魚／名人信號 2 小時半衰期；未驗證 celebrity_trade 在動態信譽路徑也會降到 social 等級，避免把未驗證喊單抬成高權威 Evidence。
  • 小螢幕 Hermes 佈局修正：HermesLeftRail、HermesRightRail 與 hermes.css 調整 rail 捲動與 1024px breakpoint，後續 codex-review 修正把 1024px 改成縮窄 rail 而不是直接隱藏，保留三欄操作脈絡。
  • 分析 composer 去重：AnalyzePage 在 embedded workspace 桌機模式隱藏中央 QueryConsole，保留左欄唯一 composer；手機因左欄隱藏，中央 QueryConsole 仍保留作唯一入口。後續 P2 修正把 CSS selector 限定在 .hermes-frame 內，避免 standalone /analyze?workspace=xxx 被誤傷。
  • AgentCore 本機開發代理路由：frontend/vite.config.ts 新增 /agentcore 與 /invocations 雙路由 proxy 到本機 127.0.0.1:8080，讓本機 UI 可測 AgentCore runtime 呼叫，但不把 AWS 或遠端環境設成預設依賴。

工程意義

這一輪把 TrustForge 的「資料可信」往兩個方向推進：一邊讓缺資料、停用來源與不安全 endpoint 明確 fail closed；另一邊新增鯨魚鏈上與名人交易作為低延遲市場參考，但用來源白名單、短半衰期、未驗證降級與測試保護，避免把高噪音訊號誤包裝成確定結論。

前端部分則是把 Hermes 艦橋從視覺完成度拉回實際可操作性：桌機不再有兩個 composer 搶焦點，小螢幕 rail 可捲動，1024px 邊界保留三欄，而 AgentCore proxy 讓本機整合測試有明確入口，不需要讓開發 UI 暗中依賴 production。
