  • Stub 稽核 allowlist：docs/audit/stub-allowlist.json 補上 trustforge.ports 的 Protocol 介面方法與 NullCacheAdapter.set 等合理 stub；同時把 AgentCoreLLMAdapter.complete/classify_stance 標成 env-gated、仍需 #637 決策的已知缺口。

工程意義

這輪不是大規模功能擴張，而是把 baseline 從「有多個已知阻塞但混在紅燈裡」整理成「哪些是真缺口、哪些是允許的介面 stub、哪些是已追蹤 xfail」的可審計狀態。對 TrustForge 的 release governance 來說，這比單純讓測試變綠更重要：它降低了假綠燈，也讓後續修 #633、#634、#636、#637 時有明確切入點。

OpenAPI 補上 /api/module-telemetry 讓 operations dashboard 與外部消費者有文件可依循；但同時保留「內部欄位還不是穩定 public schema」的限制，符合 continuous analysis 對資料 lineage 與安全邊界的要求。

前端方面，onOpenDivergence 讓右側 rail 的 divergence 操作可以回到 dashboard stage 狀態管理，也直接解除前一輪 npm run build 的 TypeScript 阻塞。這代表主線至少回到可建置狀態，但仍不能自動等同於完成 UI 眼檢、部署或 live API 驗收。
