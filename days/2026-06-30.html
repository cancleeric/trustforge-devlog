Day 1 · 2026-06-30 — 從零到「公開 Live Demo on AWS + 真實 Claude」

這是可重現日誌:照著做能得到一樣的結果。含實際指令、決策理由,以及每個踩雷與修法。環境:macOS、Python 3、已裝 gh(GitHub CLI)、git。

0. 產品決策(為什麼這樣做)

  • 命題「加密市場分析 AI Agent:多源資訊的信任提煉」。差異化押在信任層:多源資訊進 LLM 之前,先對每條主張算 TrustScore = w_src·來源信譽 + w_corr·交叉佐證 + w_rec·時效 − w_manip·操縱懲罰,輸出信任加權 + 可回溯 + 帶反方證據的報告。
  • 反作弊鐵則(攸關評分 30%):判斷/證據整合/評分由自家 pipeline 產生,LLM 只「行文」,不可把第三方現成結論當結果。
  • 對齊官方:幣種池 BTC/ETH/SOL/BNB/XRP;題型 多源整合/假設驗證/比較分析;基準資料 5 幣×5 年 Daily OHLCV CSV;限用 AWS Bedrock;執行 ≤15 分鐘。

1. 倉庫骨架與雙遠端(GitHub 主 + Gitea 鏡像)

# 專案結構

git init && git add -A && git commit -m "feat: TrustForge 奠基"
# GitHub(私有,主)
gh repo create cancleeric/trustforge --private --source=. --remote=origin --push
# Gitea 鏡像(org repo 經 API 建立後)
git remote add gitea http://<gitea-host>/hurricanesoft/trustforge.git
git push gitea main

2. 信任引擎(核心)+ 一個關鍵 bug

  • 來源信譽:price/onchain 0.95、regulatory 0.90、hoyabit 0.85、news 0.65、social 0.35。
  • 聚合門檻 0.50:高於 → 支撐證據;喊單(操縱詞)被壓到 0 → 落入反方。

踩雷① 句子切分用 . 當斷句,會把小數 46637.08、-7.4% 拆斷,丟失方向詞「下跌」。修法:只在真句末切——

re.split(r"[。！？\n]+|[.!?]+(?=\s|$)", text)   # 小數點後接數字不切

5. AWS 登入(免建 IAM key)

brew install awscli
aws configure set region ap-southeast-2
aws login           # 用瀏覽器 console session 換臨時憑證,免建 access key
aws sts get-caller-identity   # 確認

踩雷② 本機 boto3 用 aws login 憑證會要 botocore[crt];且 macOS 系統 Python 受 PEP668 管。修法:

python3 -m pip install --break-system-packages "botocore[crt]" boto3

6. Bedrock 模型存取(Anthropic use-case 閘)

  • 「Model access」頁已退役:serverless 模型首次調用自動啟用,但 Anthropic 模型首次需提交 use-case 表單(每帳號一次)。
  • 查模型 ID:aws bedrock list-inference-profiles --region ap-southeast-2 → 雪梨用 au.anthropic.claude-sonnet-4-6。
  • 查狀態:aws bedrock get-use-case-for-model-access(未填會報錯)。

踩雷③ 提交 use-case 後仍報 ResourceNotFoundException: ... try again in 15 minutes——是生效傳播延遲。修法:等 ~15 分鐘輪詢重試即可(會自動接受 Marketplace 的 Claude Sonnet 4.6 協議,$0)。

7. 佈署 A:Lambda(踩到免費方案限制)

bash deploy/deploy_lambda.sh   # 打包 zip → create/update function → Function URL

踩雷④ 函數能跑(簽章 invoke 200),但公開 Function URL 回 403,即使 auth=NONE + 公開 resource policy 都正確。真因:新帳號/免費方案擋「公開無簽章對外入口」。修法:改用 EC2(下節),或完成帳號升級。

8. 佈署 B:EC2 公開 Live Demo(成功路徑)

bash deploy/deploy_ec2.sh   # IAM 角色(最小權限)+ SG(開 80)+ S3 上傳 zip
                            # + run-instances(AL2023 t3.micro,user-data 自動裝跑)+ SSM(免金鑰)
# 輸出:http://<public-ip>/  (systemd 跑 trustforge.web)

踩雷⑤ bash 變數後緊貼中文省略號 $ROLE… 會被當變數名一部分 → unbound。修法:用 ${ROLE}… 界定。

踩雷⑥ user-data 裡 pip3 install "botocore[crt]" boto3 在 AL2023 的 awscrt 編譯失敗,連帶 boto3 沒裝,?live 直接拋 ModuleNotFoundError(表現為「回空」)。修法:EC2 走 instance role 不需 crt,改 pip3 install boto3。即時補救:

aws ssm send-command --instance-ids <id> --document-name AWS-RunShellScript \
  --parameters 'commands=["pip3 install boto3","systemctl restart trustforge"]'
# 查實例日誌也走 SSM(免 SSH key):journalctl -u trustforge / cat /var/log/tf-setup.log

9. 接通真實 Claude(EC2 上)

EC2 instance role 含 bedrock:InvokeModel + AWS_REGION + BEDROCK_MODEL_ID(systemd Environment),?live=1 即走真實 Bedrock,產出「事實層→推理鏈→結論」。離線模式(預設)則用 stub,免帳號也能展示完整信任 pipeline。

10. 安全審查 eye + codex + 即時止血

  • eye src/trustforge/trust/scoring.py:score → 耦合乾淨,無意外依賴。
  • codex 抓到 CRITICAL:公開 ?live 無驗證/限流 → 任何人可燒光 credits。

即時止血:SSM 移除實例上的 BEDROCK_MODEL_ID 並重啟 → 公開端點退回純離線(不花錢);後續派 CTO 補 token 閘 + 限流 + XSS escape + 例外處理,CISO+CPO 雙審後才重開公開 live。
